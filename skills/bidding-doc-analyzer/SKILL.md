# 招标文件智能拍案（BiddingDoc Analyzer）

## 功能

### 第一层（脚本执行）
把招标PDF转成纯文本：
- 电子档PDF → PyMuPDF直接提取文字
- 扫描件PDF → 自动降级到Tesseract OCR
- 不做任何结构化提取

### 第二层（Agent调LLM执行）
拿到文本后，Agent调用自己的LLM模型进行深度分析：

| 分析项 | 内容 | 用途 |
|--------|------|------|
| 📋 **资格条件清单** | 硬性门槛 + 加分项，分别列出 | 快速判断是否满足投标资格 |
| 📊 **评分权重表** | 价格/技术/商务分值、评分细则 | 评估中标概率，制定报价策略 |
| ⏰ **关键时间节点** | 投标截止、开标、答疑、保证金到账 | 不漏掉任何截止时间 |
| ⚠️ **异常条款提示** | 倾向性条款、排他条款、前后矛盾 | 规避陷阱，辅助质疑投诉决策 |
| 💡 **投标策略建议** | 结合评分权重给出胜算评估 | 决定投不投、怎么投 |

## 用法

```lark
@小渊 分析这份招标文件：/path/to/bidding.pdf
@小渊 看看这个标：https://example.com/procurement/doc.pdf
@小渊 帮我评标：附件是四川某安置房项目招标文件，重点看资格条件和评分权重
```

## Agent指令

拿到脚本输出的PDF文本后，按以下步骤分析：

1. **读取文本** — 从"# 招标文件原文"段落获取PDF的文字内容
2. **提取基本信息** — 项目名称、招标人、预算金额、采购方式
3. **梳理资格条件** — 分"硬性门槛"和"加分项"两类列出
4. **整理评分权重** — 价格/技术/商务分值和细则，标注异常（如"价格分占比异常低"）
5. **标记关键时间** — 投标截止、开标、保证金到账等，务必核对原文日期
6. **发现异常条款** — 倾向性条款（如"指定某品牌"）、排他条款（如"要求某资质兼有"）、前后矛盾
7. **给出策略建议** — 结合评分权重和自身条件给出"投/不投"判断

### 输出格式
```markdown
📋 资格条件清单
【硬性门槛】
1. ...
【加分项】
1. ...

📊 评分权重表
价格分：XX分 | 技术分：XX分 | 商务分：XX分

⏰ 关键时间节点
- 投标截止：YYYY-MM-DD HH:mm

⚠️ 异常条款提示
...

💡 投标策略建议
...
```

### 注意事项
- 所有结论标注置信度（高/中/低）
- 模糊不清的条款标注"需人工复核"
- 不预设任何省份的招标格式，通用解析
- 投标决策以招标文件原文为准

## 文件说明

- `SKILL.md` — 技能定义文件
- `bidding_doc_analyzer.py` — 核心分析脚本

## 安装依赖

```bash
# 系统依赖（OCR中文支持）
sudo apt-get install -y tesseract-ocr tesseract-ocr-chi-sim
# Python依赖
pip install pymupdf pytesseract
```

## 注意事项

- 支持扫描件PDF（自动启用OCR）和电子档PDF
- 扫描件OCR准确率受原文件清晰度影响，关键条款建议人工复核
- 同一份文件可重复分析，结果缓存命中时直接返回
- 分析结果仅作参考，投标决策请以招标文件原文为准
