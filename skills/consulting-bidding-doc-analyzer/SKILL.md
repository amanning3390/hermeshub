# 招标文件智能拍案（BiddingDoc Analyzer）

## When to Use

- User uploads or provides a PDF of a bidding/tender document
- User asks "帮我看看这个标" or "分析这份招标文件"
- User needs qualification requirements, scoring criteria, or key dates extracted from a procurement document
- User wants an assessment of bid competitiveness and winning probability
- User suspects unfair or contradictory clauses in a tender document

## 功能

### 第一层（脚本执行）
把招标PDF转成纯文本：
- 电子档PDF → PyMuPDF直接提取文字
- 扫描件PDF → 自动降级到Tesseract OCR
- 不做任何结构化提取

### 第二层（Agent调LLM执行）
拿到文本后，Agent调用自己的LLM模型进行深度分析：

| 分析项 | 内容 | 用途 |
|--------|------|------|
| 📋 **资格条件清单** | 硬性门槛 + 加分项，分别列出 | 快速判断是否满足投标资格 |
| 📊 **评分权重表** | 价格/技术/商务分值、评分细则 | 评估中标概率，制定报价策略 |
| ⏰ **关键时间节点** | 投标截止、开标、答疑、保证金到账 | 不漏掉任何截止时间 |
| ⚠️ **异常条款提示** | 倾向性条款、排他条款、前后矛盾 | 规避陷阱，辅助质疑投诉决策 |
| 💡 **投标策略建议** | 结合评分权重给出胜算评估 | 决定投不投、怎么投 |

## Procedure

### 第一步：验证和准备
1. 确认用户提供了PDF文件路径或URL
2. 调用脚本 `python3 bidding_doc_analyzer.py <path>` 提取文本
3. 检查脚本输出：如果输出错误信息（Tesseract未安装、文件损坏等），直接报告错误给用户
4. 如果提取的文本少于50个字符，提示"文档可能为空或不可读，请检查PDF文件"

### 第二步：分析（Agent调用LLM）
拿到脚本输出的文本后，按顺序执行：

1. **读取文本** — 从脚本输出的文本中获取PDF的全部文字内容
2. **提取基本信息** — 项目名称、招标人、预算金额、采购方式
3. **梳理资格条件** — 分"硬性门槛"和"加分项"两类列出，标注每条置信度
4. **整理评分权重** — 价格/技术/商务分值和细则，标注异常（如"价格分占比异常低"）
5. **标记关键时间** — 投标截止、开标、保证金到账等，务必核对原文日期
6. **发现异常条款** — 倾向性条款、排他条款、前后矛盾
7. **给出策略建议** — 结合评分权重和自身条件给出"投/不投"判断

### 第三步：输出
按下方要求的结构化模板输出结果。

## Pitfalls

- **不要编造原文没有的内容** — 如果招标文件没有明确写出某项信息（如评分细则），标注"原文未提供"而不是自己编
- **不要猜日期** — 时间节点必须原文有据可查，模糊的日期（如"约7月中旬"）标注置信度低
- **不要写具体中标概率数字** — 策略建议用文字描述胜算，不要写"中标概率70%"
- **不要预设省份格式** — 全国通用，不假设四川/广东/北京等特定格式
- **OCR文字可能错** — 扫描件识别结果中的关键条款（资质要求、金额、日期）全部标注置信度，模糊的标"需人工复核"
- **前30页截断** — 超过30页的文件自己不知道"后面还有内容"，必须在末尾明确提醒用户

## Verification

- [ ] 资格条件清单的每一条都能在原文中找到对应文字
- [ ] 评分权重表的各项分值加总等于100分（如有明确说明）
- [ ] 所有时间节点在原文中确有其文，不是自己推算的
- [ ] 异常条款标注了严重度等级
- [ ] 策略建议标明"建议"而非结论性断言
- [ ] 置信度标注完整，无遗漏
- [ ] OCR识别的内容全部标注了置信度
- [ ] 超过30页的文件有截断提示

## 注意事项

- 支持扫描件PDF（自动启用OCR）和电子档PDF
- 扫描件OCR准确率受原文件清晰度影响，关键条款建议人工复核
- 同一份文件可重复分析，结果缓存命中时直接返回
- 分析结果仅作参考，投标决策请以招标文件原文为准

## 已知限制

- **扫描件OCR精度**：扫描件PDF的OCR准确率受原稿清晰度影响（300DPI渲染），模糊/折痕/印章遮挡区域可能出现误识别
- **页数限制**：超过30页的招标文件自动截断前30页分析，完整分析需分批手动处理
- **Tesseract依赖**：扫描件识别依赖本地Tesseract安装，未安装时自动跳过OCR（电子档PDF不受影响）
- **非标准格式**：部分招标文件使用图片式表格或手写标注，可能影响提取精度
- **数据安全**：PDF文件在本地处理，不会上传到任何第三方服务
